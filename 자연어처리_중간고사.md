### 1. 텍스트 전처리 (02_텍스트_전처리)
비정형 데이터를 기계가 분석할 수 있는 정형 데이터로 변환하는 필수 과정입니다.

| 핵심 항목 | 상세 메커니즘 및 설명 | 특징 및 주의사항 |
| :--- | :--- | :--- |
| **텍스트 마이닝** | 비정형 텍스트 → 패턴/트렌드 분석 → 고품질 정보 추출 과정 | 데이터 마이닝과 달리 '언어적 해석' 포함 |
| **정규화 (Normalization)** | **어간 추출(Stemming)**: 규칙 기반으로 단어 어미 제거 (예: cats→cat, flies→fli) <br> **표제어 추출(Lemmatization)**: 품사 정보를 고려하여 사전적 원형 복원 (예: am, are→be) | Lemmatization이 Stemming보다 정교하지만 속도는 느림 |
| **한국어 전처리** | **교착어 성질**: '나(대명사) + 는(조사)' 처럼 의미 단위인 **형태소** 분리가 필수 <br> **동음이의어**: '눈(eye/snow)' 처럼 형태는 같으나 의미가 다른 경우 **품사 태깅(POS Tagging)**으로 문맥 파악 | 형태소 분석기(Mecab, Okt 등) 활용 필수 |
| **Zipf의 법칙** | 극소수의 단어가 전체 빈도의 대부분을 차지하고, 대다수의 단어는 빈도가 매우 낮음 | 데이터 희소성(Sparsity) 문제의 근거 |

### 2. 카운트 기반 문서 표현 (03_카운트기반_문서_표현)
단어의 빈도를 활용하여 문서를 고차원 벡터 공간의 점으로 수치화합니다.

| 핵심 항목 | 상세 수식 및 계산법 | 메커니즘의 의미 |
| :--- | :--- | :--- |
| **코사인 유사도** | $\cos(\theta) = \frac{A \cdot B}{\|A\|\|B\|}$ | 벡터의 크기(문장 길이)보다 **'방향(단어 구성 비중)'**에 집중 |
| **BoW / DTM** | 단어의 순서는 무시하고 오직 빈도(Count)만 기록하여 행렬로 표현 | 문맥 정보가 소실되며, 행렬의 대부분이 0인 **희소 행렬** 생성 |
| **TF-IDF 계산** | $TF \text{-} IDF(t, d) = tf(t, d) \times \log\frac{N}{df(t)}$ | **TF**: 문서 내 빈도 / **IDF**: 단어가 등장한 문서 수의 역수 <br> 관사(the, a) 등 흔한 단어 가중치는 낮추고 핵심어 강조 |
| **유클리디안 거리** | $d(x,y) = \sqrt{\sum (x_i - y_i)^2}$ | 두 점 사이의 직선 거리 측정 (값이 작을수록 유사) |

### 3. 문서 분류 및 성능 지표 (04_Bow기반의 문서 분류)
과거 데이터를 학습하여 새로운 문서의 카테고리를 결정하는 지도학습 기법입니다.

| 핵심 항목 | 상세 메커니즘 및 수식 | 모델 특징 및 보정 |
| :--- | :--- | :--- |
| **나이브 베이즈** | $P(C|x) = \frac{P(x|C)P(C)}{P(x)}$ <br> 모든 단어가 독립적으로 발생한다고 가정하고 확률을 곱함 | 계산이 매우 빠르며 텍스트 분류에서 성능이 준수함 |
| **N-gram** | $n$개의 연속된 단어를 하나의 단위로 취급 (Unigram, Bigram, Trigram) | 'not happy'처럼 단어 조합의 의미(문맥)를 일부 보존 가능 |
| **희소성 문제 해결** | **Laplace Smoothing**: 빈도가 0인 단어 확률에 1을 더해 확률이 0이 되는 현상 방지 <br> **Back-off**: $n$차 N-gram에 데이터가 없으면 $n-1$차로 내려가 계산 | 실제 데이터가 없는 상황에 대비하는 통계적 기법 |
| **성능 지표 (F1)** | **정밀도(P)**: 모델이 참이라 한 것 중 실제 참 <br> **재현율(R)**: 실제 참 중 모델이 맞춘 비율 <br> **F1-Score**: $\frac{2 \times P \times R}{P + R}$ | 데이터가 불균형할 때(예: 스팸 1건 vs 일반 99건) 정확도보다 신뢰할 수 있음 |

### 4. 차원 축소 및 군집화 (05_차원_축소_및_군집분류)
고차원 데이터의 중복을 제거하고(차원 축소) 정답 없이 유사성을 그룹화(군집화)합니다.

| 핵심 항목 | 상세 메커니즘 및 지표 | 핵심 원리 |
| :--- | :--- | :--- |
| **차원의 저주** | 차원이 늘어날수록 공간이 희박해져 데이터 간 거리가 멀어지고 성능이 저하되는 현상 | 데이터 증가량보다 공간의 팽창이 훨씬 빠름 |
| **LSA / SVD** | $A = U \Sigma V^T$ (특잇값 분해) <br> 문서-단어 행렬을 축소하여 단어들 사이의 **잠재된 의미** 추출 | 유의어(거의 같은 의미의 다른 단어) 관계 파악에 효과적 |
| **K-평균 군집화** | 각 데이터로부터 중심점까지의 거리를 계산하여 가까운 군집에 할당 | 거리 기반 비지도 학습의 대표 알고리즘 |
| **ARI (평가 지표)** | **Adjusted Rand Index**: 우연히 두 데이터가 같은 군집에 속할 확률을 보정한 지표 | 1에 가까울수록 실제 정답과 군집 결과가 일치함 |

### 5. 토픽 모델링 및 감성 분석 (06_토픽 모델링 및 감성분석)
텍스트의 추상적인 주제를 발견하고 주관적인 감성 상태를 분석합니다.

| 핵심 항목 | 상세 메커니즘 및 지표 | 분석 포인트 |
| :--- | :--- | :--- |
| **LDA (Topic Modeling)** | 문서는 토픽의 혼합체이며, 토픽은 단어의 확률 분포라는 가정 하에 **디리클레 분포** 사용 | 사후 분석(이미 있는 데이터의 의미 파악)에 최적화 |
| **Perplexity (혼란도)** | 모델이 데이터를 얼마나 잘 예측하는지 측정. **낮을수록 우수** | 확률 모델의 언어 이해 성능 지표 |
| **Coherence (응집도)** | 하나의 토픽에 묶인 단어들이 얼마나 의미적으로 유사한지 측정. **높을수록 우수** | 인간이 이해하기에 얼마나 일관성 있는 주제인가를 판단 |
| **감성 분석 극성** | **Polarity**: -1(부정) ~ 0(중립) ~ +1(긍정) 범위의 수치로 감정 강도 표현 | 텍스트의 주관적 정보를 정량화 |

### 6. 인공신경망과 딥러닝 (07_인공신경망과 딥러닝의 이해)
인간의 뉴런을 모사한 구조로 복잡한 비선형 관계를 학습합니다.

| 핵심 항목 | 상세 메커니즘 및 수식 | 학습 원리 및 한계 해결 |
| :--- | :--- | :--- |
| **뉴런 연산** | **가중합**: $z = \sum w_i x_i + b$ <br> **활성화 함수**: $y = f(z)$ (Sigmoid, ReLU 등) | 입력값에 중요도(가중치)를 곱하고 편향을 더해 신호 전달 |
| **오차 역전파** | 출력층의 오차를 뒤쪽 층으로 전달하며 **연쇄 법칙(Chain Rule)**을 통해 가중치 수정 | 오차를 줄이는 방향으로 모든 가중치를 동시에 업데이트 |
| **경사 하강법** | $w_{t+1} = w_t - \eta \frac{\partial L}{\partial w}$ ($\eta$: 학습률) | 손실 함수(Loss)의 기울기가 0이 되는 지점(최솟값)을 찾아 이동 |
| **ReLU 함수** | $f(x) = \max(0, x)$ | 양수일 때 기울기가 1로 유지되어 **기울기 소실(Vanishing Gradient)** 문제 완화 |
| **최적화 기법** | **Adam, SGD, Momentum**: 지역 최솟값(Local Minimum)에 빠지지 않고 전역 최솟값을 찾기 위한 전략 | 학습 속도와 정확도를 동시에 고려한 알고리즘 |
