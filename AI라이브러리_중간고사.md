# [강의 요약] 인공지능 라이브러리 - 01. 기초 개념 및 파이토치

**강사:** 백찬은 교수 (동양미래대학교 인공지능소프트웨어학과)
**주교재:** 딥러닝의 정석 with 파이토치 (길벗캠퍼스)

---

## 1. 인공지능 개요 (AI vs ML vs DL)

### 1.1 인공지능 (AI)
- 인간의 지능적인 작업을 컴퓨터 시스템이 수행하도록 하는 기술 및 학문 분야.
- **예시:** 챗봇(ChatGPT), 자율 주행, 음성 인식 등.

### 1.2 머신러닝 (ML)
- 명시적인 프로그래밍 없이 데이터에서 패턴을 학습하여 예측이나 결정을 내리는 기술.
- **핵심:** 사람이 직접 특징(Feature)을 설계하여 모델에 전달함.

### 1.3 딥러닝 (DL)
- 인간의 뇌 구조를 모방한 **인공신경망(ANN)**을 사용하여 복잡한 패턴을 학습.
- **머신러닝과의 차이점:**
  - **특징 추출:** 모델이 데이터로부터 직접 특징을 추출 (Feature Engineering 과정 축소).
  - **요구 사항:** 대량의 데이터와 강력한 연산 능력(GPU)이 필요함.

---

## 2. 학습 방식의 분류

| 분류 | 특징 | 주요 사례 |
| :--- | :--- | :--- |
| **지도학습 (Supervised)** | 정답(Label)이 있는 데이터를 학습 | 스팸 메일 분류, 주택 가격 예측 |
| **비지도학습 (Unsupervised)** | 정답 없이 데이터의 구조나 패턴 발견 | 고객 세분화(Clustering), 이상 탐지 |
| **강화학습 (Reinforcement)** | 행동에 따른 보상을 통해 최적의 전략 학습 | 게임 AI(알파고), 로봇 제어 |

---

## 3. 기초 통계 및 수학 지식

- **변수의 종류**
  - **독립변수 (Independent):** 원인이 되는 변수 (입력값).
  - **종속변수 (Dependent):** 결과가 되는 변수 (출력값).
- **데이터의 분포**
  - **평균 (Mean):** 데이터의 중심 위치를 나타냄.
  - **분산 (Variance):** 데이터가 평균으로부터 얼마나 퍼져 있는지를 나타냄.

---

## 4. 파이토치 (PyTorch) 및 계산 그래프

### 4.1 파이토치의 특징
- 파이썬 기반의 오픈소스 머신러닝 라이브러리.
- 직관적인 문법으로 연구와 배포가 용이함.
- **GPU 가속**을 지원하여 빠른 연산 가능.

### 4.2 계산 그래프 (Computation Graph)
딥러닝 프레임워크는 연산 과정을 그래프로 표현합니다.

- **정적 계산 그래프 (Static):**
  - 그래프를 미리 정의한 후 데이터를 입력 (TensorFlow 1.x).
  - 실행 전 최적화가 가능하여 속도가 빠르지만 유연성이 떨어짐.
- **동적 계산 그래프 (Dynamic - PyTorch):**
  - **Define by Run:** 연산이 실행될 때마다 그래프가 생성됨.
  - 파이썬 제어문(if, for)을 자유롭게 사용 가능하며 디버깅이 매우 쉬움.

# [강의 요약] 인공지능 라이브러리 - 03. 신경망 구현 및 학습 원리

**강사:** 백찬은 교수 (동양미래대학교 인공지능소프트웨어학과)
**핵심 주제:** 신경망 구조, 순전파/역전파, 손실함수, 경사하강법, 옵티마이저

---

## 1. 신경망과 인공 뉴런

### 1.1 생물학적 뉴런 vs 인공 뉴런
- **생물학적 뉴런:** 수상돌기(입력) -> 축삭(전달) -> 시냅스(출력)의 과정을 거침.
- **인공 뉴런 (Perceptron):** 입력값($x$)에 가중치($w$)를 곱하고 편향($b$)을 더한 뒤, **활성화 함수**를 통과시켜 최종 출력을 결정함.


### 1.2 인공 신경망의 발전
- **1940s:** 최초의 수학적 뉴런 제안 (AND, OR 연산).
- **1950s:** 퍼셉트론(Perceptron) 개발 (선형 분류 가능, XOR 문제 해결 불가).
- **1980s:** 다층 퍼셉트론(MLP)과 **역전파(Backpropagation)** 알고리즘 도입으로 비선형 문제 해결.

---

## 2. 신경망의 핵심 구성 요소

### 2.1 신경망의 층(Layer) 구조
1. **입력층 (Input Layer):** 외부 데이터를 받아들이는 층.
2. **은닉층 (Hidden Layer):** 데이터의 특징을 추출하는 중간 층 (깊을수록 딥러닝).
3. **출력층 (Output Layer):** 최종 예측 결과를 내보내는 층.

### 2.2 활성화 함수 (Activation Function)
- 선형 결합된 결과에 **비선형성**을 부여하여 복잡한 패턴을 학습하게 함.
- **주요 종류:**
  - **Sigmoid:** 0~1 사이 출력 (이진 분류).
  - **ReLU:** 0보다 작으면 0, 크면 그대로 출력 (가장 많이 사용됨).
  - **Softmax:** 다중 클래스 분류 시 각 클래스별 확률 출력.

---

## 3. 모델 학습 원리

### 3.1 순전파 (Forward Propagation)
- 입력 데이터를 층별로 통과시켜 예측값(Output)을 계산하는 과정.

### 3.2 손실 함수 (Loss Function)
- 예측값과 실제 정답(Label) 간의 차이를 수치화함.
- **MSE (Mean Squared Error):** 회귀 문제에서 사용.
- **Cross-Entropy:** 분류 문제에서 사용.

### 3.3 역전파 (Backpropagation)
- 출력층에서 발생한 오차를 입력층 방향으로 거슬러 올라가며 전달하여 각 층의 **가중치를 얼마나 수정할지(기울기)** 결정함.


---

## 4. 경사하강법 및 옵티마이저

### 4.1 경사하강법 (Gradient Descent)
- 손실 함수의 값을 최소화하기 위해 기울기(Gradient)의 반대 방향으로 가중치를 조금씩 업데이트하는 방법.

### 4.2 경사하강법의 종류
- **BGD (Batch Gradient Descent):** 전체 데이터를 보고 업데이트 (느림).
- **SGD (Stochastic Gradient Descent):** 데이터 한 개마다 업데이트 (빠르지만 불안정).
- **Mini-Batch GD:** 데이터를 일정 묶음(Batch)으로 나누어 학습 (속도와 안정성 모두 확보, 실무 권장).

### 4.3 옵티마이저 (Optimizer)
- 경사하강법을 더 빠르고 정확하게 수행하도록 개선한 알고리즘.
- **Adam:** 방향(Momentum)과 보폭(RMSProp)의 장점을 합친 가장 대중적인 알고리즘.

---

## 5. 신경망 구현 단계 (PyTorch workflow)
1. **데이터 준비:** `Dataset`, `DataLoader` 정의.
2. **모델 정의:** `nn.Module` 상속 후 구조 설계.
3. **손실 함수 및 옵티마이저 설정.**
4. **학습 루프:** `Forward` -> `Loss` 계산 -> `Backward` -> `Step` (가중치 업데이트).

# [강의 요약] 인공지능 라이브러리 - 04. 모델 학습 과정과 성능 개선

**강사:** 백찬은 교수 (동양미래대학교 인공지능소프트웨어학과)
**핵심 주제:** 학습의 상세 단계, DataLoader 활용, 모델 평가 및 하이퍼파라미터 튜닝

---

## 1. 모델의 4단계 학습 과정

모델이 데이터를 통해 학습하는 과정은 크게 다음 네 단계로 반복됩니다.

1. **순전파 (Forward Pass):** - 입력 데이터가 계층별 연산(선형 변환 및 활성화 함수)을 거쳐 예측값을 생성하는 과정.
2. **손실 계산 (Loss Calculation):** - 예측값과 실제 정답의 차이를 손실 함수(MSE, Cross-Entropy 등)로 수치화.
3. **역전파 (Backward Pass):** - 출력에서 입력 방향으로 손실에 대한 기울기(Gradient)를 계산.
4. **파라미터 업데이트 (Parameter Update):** - 옵티마이저를 사용하여 계산된 기울기 방향으로 가중치($W$)와 편향($b$)을 수정.

---

## 2. 데이터 전처리와 DataLoader

효율적인 학습을 위해 파이토치에서 제공하는 데이터 관리 도구입니다.

### 2.1 Dataset과 DataLoader
- **Dataset:** 전체 데이터를 저장하고, 인덱스를 통해 개별 데이터를 반환하는 역할.
- **DataLoader:** Dataset을 래핑하여 데이터를 미니 배치(Mini-batch) 단위로 묶어 모델에 전달.

### 2.2 주요 기능
- **Batch Size:** 한 번에 처리할 샘플의 개수 (예: 64). GPU의 병렬 연산을 극대화함.
- **Shuffle:** 매 에포크(Epoch)마다 데이터 순서를 섞어 모델의 일반화 성능을 향상.
- **Num_workers:** 멀티프로세싱을 통해 데이터 로딩 속도를 높여 I/O 병목 현상 방지.

---

## 3. 모델 성능 평가 및 개선

### 3.1 하이퍼파라미터 튜닝 (Hyperparameter Tuning)
모델의 가중치처럼 학습으로 결정되는 것이 아니라, 사용자가 직접 설정해야 하는 변수들입니다.
- **종류:** 학습률(Learning Rate), 배치 크기(Batch Size), 에포크(Epoch) 수 등.
- **탐색 방법:**
  - **Grid Search:** 미리 정해진 조합을 모두 시도 (정교하지만 오래 걸림).
  - **Random Search:** 무작위로 조합을 선택하여 탐색.
  - **Bayesian Optimization:** 이전 결과를 바탕으로 최적의 조합을 추론.



### 3.2 모델 평가 지표
- **분류(Classification):** 정확도(Accuracy), 정밀도(Precision), 재현율(Recall), F1-Score.
- **회귀(Regression):** MSE(평균 제곱 오차), MAE(평균 절대 오차).

---

## 4. 과적합(Overfitting)과 과소적합(Underfitting)

- **과적합 (Overfitting):** 모델이 학습 데이터에 너무 과하게 맞춰져 실제 데이터(검증 데이터)에서 성능이 떨어지는 현상.
- **과소적합 (Underfitting):** 모델이 너무 단순하여 학습 데이터의 패턴조차 제대로 파악하지 못한 상태.
- **해결책:** 데이터 증강(Augmentation), 드롭아웃(Dropout), 조기 종료(Early Stopping) 등 적용.



---

## 5. 종합 실습 흐름
1. 파이토치를 이용한 **신경망 모델 설계** (`nn.Module`).
2. **DataLoader**를 통한 데이터 공급.
3. **학습 루프** 구현 및 **손실값 변화** 모니터링.
4. **테스트 데이터**를 활용한 최종 모델 성능 검증.

# [강의 요약] 인공지능 라이브러리 - 05. 합성곱 신경망 (CNN) Part 1

**강사:** 백찬은 교수 (동양미래대학교 인공지능소프트웨어학과)
**핵심 주제:** CNN의 필요성, 합성곱(Convolution) 원리, 주요 레이어(Padding, Stride, Pooling)

---

## 1. CNN의 등장 배경

### 1.1 완전 연결 신경망(FCNN)의 한계
- **공간 정보 소실:** 이미지를 1차원 벡터로 펼쳐서 처리하므로, 픽셀 간의 위치 관계(상하좌우) 정보가 사라짐.
- **파라미터 폭증:** 이미지 크기가 커질수록 연산해야 할 가중치가 기하급수적으로 늘어나 학습이 어렵고 과적합 위험이 높음.
  - 예: $224 \times 224 \times 3$ 이미지 입력 시 입력층 뉴런만 150,528개 필요.

### 1.2 CNN의 장점
- 이미지의 **2D 구조(공간적 정보)**를 유지하면서 특징을 추출.
- **가중치 공유(Weight Sharing):** 작은 필터를 반복 사용하여 학습 파라미터 수를 획기적으로 줄임.

---

## 2. CNN의 주요 구성 요소

### 2.1 합성곱 레이어 (Convolutional Layer)
- **커널/필터 (Kernel/Filter):** 이미지 위를 슬라이딩하며 특징을 찾아내는 작은 행렬.
- **합성곱 연산:** 필터와 이미지의 겹치는 부분의 요소별 곱을 모두 더해 하나의 값을 도출.
- **특징 맵 (Feature Map):** 합성곱 연산을 통해 얻어진 결과물(추출된 이미지 특징).



### 2.2 패딩(Padding)과 스트라이드(Stride)
- **스트라이드 (Stride):** 필터가 한 번에 이동하는 칸수. 스트라이드가 커지면 출력 특징 맵의 크기는 작아짐.
- **패딩 (Padding):** 출력 크기가 줄어드는 것을 방지하기 위해 입력 이미지 테두리에 0 등을 채우는 기법.
  - **Valid Padding:** 패딩 없음.
  - **Same Padding:** 출력 크기를 입력과 동일하게 유지.



### 2.3 풀링 레이어 (Pooling Layer)
- **정의:** 특징 맵의 크기를 줄여 연산량을 감소시키고 주요 특징을 강조하는 과정 (다운샘플링).
- **종류:**
  - **Max Pooling:** 설정된 영역 내에서 가장 큰 값을 선택 (가장 많이 사용).
  - **Average Pooling:** 영역 내 평균값을 계산.

---

## 3. CNN의 전체 구조 (Architecture)

1. **특징 추출부 (Feature Extraction):** - Convolution 레이어 + 활성화 함수(ReLU) + Pooling 레이어를 반복하며 이미지 특징 파악.
2. **분류기 (Classification):** - 추출된 특징을 1차원으로 펼친 뒤(**Flatten**), 완전 연결 층(FC Layer)을 거쳐 최종 클래스 분류.



---

## 4. 학습 팁
- **필터의 수:** 각 레이어에서 추출할 특징의 개수를 결정함.
- **수용 영역 (Receptive Field):** 출력부의 한 픽셀이 입력 이미지의 어느 정도 범위를 보고 있는지 나타내는 개념.

# [강의 요약] 인공지능 라이브러리 - 06. CNN 심화 및 모델 최적화

**강사:** 백찬은 교수 (동양미래대학교 인공지능소프트웨어학과)
**핵심 주제:** VGGNet, ResNet(잔차 학습), 모델 최적화(경량화, 양자화, 지식 증류)

---

## 1. 대표적인 CNN 아키텍처

### 1.1 VGGNet
- **특징:** 모든 합성곱 층에서 **3x3 크기의 작은 커널**을 사용하며 층을 깊게 쌓은 것이 특징입니다.
- **구조:** 층의 개수에 따라 VGG16, VGG19 등으로 나뉩니다.
- **장점:** 구조가 단순하여 이해하기 쉽고, 작은 필터를 여러 번 쌓아 비선형성을 증대시키면서 파라미터 수를 효율적으로 관리합니다.

### 1.2 ResNet (Residual Network)
- **배경:** 네트워크가 깊어질수록 오히려 학습이 어려워지는 **기울기 소실(Vanishing Gradient)** 문제를 해결하기 위해 등장했습니다.
- **핵심 기술: 잔차 학습(Residual Learning)**
  - 입력값($x$)을 출력값에 직접 더해주는 **지름길(Shortcut Connection/Skip Connection)**을 만듭니다.
  - 모델이 '입력과 정답 사이의 차이(잔차)'만 학습하게 하여 100층 이상의 매우 깊은 망도 안정적으로 학습할 수 있게 합니다.



---

## 2. 모델 최적화 및 경량화 (Optimization)

학습된 모델을 실제 기기(스마트폰, 임베디드 등)에서 효율적으로 실행하기 위한 기법들입니다.

### 2.1 가지치기 (Pruning)
- 중요도가 낮은(값이 작은) 가중치나 필터를 제거하여 모델의 크기를 줄이는 기법입니다.
- **장점:** 파라미터 수가 줄어들어 추론 속도가 빨라지고 메모리가 절약됩니다.

### 2.2 양자화 (Quantization)
- 모델의 가중치를 높은 정밀도(예: 32비트 실수)에서 낮은 정밀도(예: 8비트 정수)로 변환하는 기법입니다.
- **장점:** 모델 크기가 약 1/4로 줄어들며, 하드웨어 연산 속도가 향상됩니다.



### 2.3 지식 증류 (Knowledge Distillation)
- 미리 잘 학습된 거대한 모델(**Teacher**)의 지식을 작고 효율적인 모델(**Student**)에게 전달하는 학습 방식입니다.
- **장점:** 작은 모델이 큰 모델의 성능을 유사하게 따라갈 수 있게 하여 경량화를 달성합니다.

---

## 3. 요약 및 성능 비교
- **네트워크 깊이:** ResNet > VGGNet 순으로 깊은 구조를 가집니다.
- **최적화 필요성:** 고성능 모델일수록 연산량이 많으므로, 실무 배포 시에는 반드시 가지치기나 양자화 같은 최적화 과정이 수반됩니다.

# [강의 요약] 인공지능 라이브러리 - 07. 순환 신경망 (RNN, LSTM, GRU)

**강사:** 백찬은 교수 (동양미래대학교 인공지능소프트웨어학과)
**핵심 주제:** 시퀀스 데이터 처리, RNN의 구조와 한계, LSTM 및 GRU의 게이트 매커니즘

---

## 1. RNN (Recurrent Neural Network)

### 1.1 개요
- **정의:** 순서가 있는 데이터(시퀀스 데이터)를 처리하기 위해 내부적으로 순환하는 구조를 가진 신경망입니다.
- **특징:** 이전 시점($t-1$)의 정보를 현재 시점($t$)의 입력과 함께 처리하여 '기억'을 유지합니다.
- **적용 분야:** 문장(자연어), 음성, 주가, 센서 데이터 등 시간의 흐름에 따라 의미가 변하는 데이터.

### 1.2 수식 및 구조
- $h_t = f(W \cdot x_t + U \cdot h_{t-1})$
  - $x_t$: 현재 입력
  - $h_{t-1}$: 이전 은닉 상태 (Hidden State)
  - $W, U$: 모든 시점에서 **공유되는 가중치**

### 1.3 한계점 (장기 의존성 문제)
- 입력 시퀀스가 길어질수록 앞부분의 정보가 뒤로 전달되지 않는 **기울기 소실(Vanishing Gradient)** 현상이 발생합니다.

---

## 2. LSTM (Long Short-Term Memory)

RNN의 장기 의존성 문제를 해결하기 위해 **Cell State**와 **Gate(게이트)** 구조를 도입한 모델입니다.

### 2.1 주요 구조

- **Cell State ($c_t$):** 정보를 장기적으로 보존하는 '컨베이어 벨트' 역할.
- **Forget Gate:** 이전 정보 중 무엇을 버릴지 결정.
- **Input Gate:** 새로운 정보 중 무엇을 저장할지 결정.
- **Output Gate:** 어떤 정보를 다음 은닉 상태로 보낼지 결정.

### 2.2 장점
- 중요한 정보는 오래 기억하고, 불필요한 정보는 삭제함으로써 긴 시퀀스에서도 성능을 유지합니다.

---

## 3. GRU (Gated Recurrent Unit)

### 3.1 개요
- LSTM의 복잡한 구조를 단순화하여 학습 속도를 개선한 모델입니다.
- **특징:** Cell State와 Hidden State를 하나로 합쳤으며, 게이트 수를 2개로 줄였습니다.

### 3.2 주요 게이트
- **Update Gate:** 과거와 현재의 정보 중 어느 정도 비율로 업데이트할지 결정.
- **Reset Gate:** 과거의 정보를 어느 정도 무시할지 결정.

---

## 4. 모델별 비교 요약

| 모델 | 특징 | 장점 | 단점 |
| :--- | :--- | :--- | :--- |
| **RNN** | 가장 기본적인 순환 구조 | 구조가 단순하고 빠름 | 장기 의존성 문제(기울기 소실) |
| **LSTM** | 3개의 게이트 + Cell State | 긴 시퀀스 학습에 강함 | 구조가 복잡하고 연산량이 많음 |
| **GRU** | 2개의 게이트 (단순화) | LSTM과 유사한 성능, 빠른 학습 | 데이터 양에 따라 LSTM보다 불리할 수 있음 |

---

## 5. 실습 내용
- 파이토치의 `nn.RNN`, `nn.LSTM`, `nn.GRU` 모듈 사용법.
- `hidden_size`, `num_layers`, `batch_first` 등 주요 파라미터 설정.
- 시퀀스 데이터를 텐서 형태로 변환하여 모델에 입력하는 과정.
