# 📑 Streamlit & AI 모델 활용 교육 과정 상세 요약

## 1. Streamlit 소개 및 핵심 원리
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | Streamlit의 철학 이해 및 **Data Flow(재실행)** 원리 습득 |
| **핵심 이론** | - **Data Flow**: 위젯 조작 시 스크립트 전체가 상단부터 하단까지 재실행됨<br>- **Magic Commands**: 변수명만 작성해도 `st.write()` 처럼 화면에 출력 |
| **주요 기능** | - `st.write()`: 데이터프레임, 차트, 텍스트 등을 자동 인식하여 출력<br>- `st.title()`, `st.header()`, `st.markdown()`: 웹 UI 텍스트 구성 |
| **실무 팁** | HTML/CSS 없이 Python만으로 상호작용 가능한 데이터 앱 제작에 최적 |

## 2. 캐싱 실전, 세션 상태 심화, 폼
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | 성능 최적화를 위한 **캐싱**과 사용자 상태 유지 기법 마스터 |
| **캐싱 전략** | - **@st.cache_data**: API 호출, CSV 로딩 등 '값'을 리턴하는 함수에 사용<br>- **@st.cache_resource**: AI 모델 객체, DB 연결 등 '리소스' 유지에 사용 |
| **상태 관리** | - `st.session_state`: 페이지 재실행 시에도 초기화되지 않는 변수 저장소<br>- `st.form`: 여러 위젯 값을 모아 Submit 시 한 번에 처리 (재실행 최소화) |
| **주의 사항** | `st.set_page_config()`는 반드시 코드의 가장 첫 번째 줄에 위치해야 함 |

## 4. Hugging Face 소개 및 Pipeline 기초
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | AI 모델의 GitHub인 **Hugging Face** 생태계 이해 및 기초 실습 |
| **핵심 플랫폼** | - **Models**: 100만 개 이상의 사전 학습 모델 공유<br>- **Datasets**: 데이터셋 허브 / **Spaces**: AI 데모 웹 서비스 공간 |
| **Pipeline API** | 복잡한 딥러닝 내부 구조 없이 `pipeline("task")` 호출로 AI 기능 바로 사용 |
| **내부 원리** | 입력 텍스트 → **Tokenizer**(숫자 변환) → **Model**(연산/추론)의 결합 구조 |

## 5. Pipeline 심화, Tokenizer & Model
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | 다양한 NLP 태스크 확장 및 **한국어 특화 모델** 적용법 학습 |
| **주요 태스크** | - **제로샷 분류**: 학습 데이터 없이도 즉석에서 카테고리 분류 가능<br>- **요약/번역**: `summarization`, `translation` 태스크 실습 |
| **한국어 최적화** | `model="digit82/kobart-summarization"` 등 특정 모델 지정 필수 |
| **학습 포인트** | 토큰화 → 모델 예측 → 후처리(Post-processing)로 이어지는 단계별 동작 이해 |

## 6. Ollama 소개 및 로컬 LLM 기초
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | 보안과 비용 걱정 없는 **로컬 기반 거대 언어 모델(LLM)** 환경 구축 |
| **주요 파라미터** | - **Temperature**: 0에 가까우면 정답 위주, 1에 가까우면 창의적 답변 생성<br>- **Token**: 모델이 텍스트를 인식하고 처리하는 최소 단위 |
| **API 활용** | `ollama.chat()`: **system**(역할 부여), **user**(질문), **assistant**(답변) 구조 활용 |
| **차이점** | 5주차 전문 모델(Pipeline)은 특정 도구, 6주차 LLM은 범용적인 지능형 비서 역할 |

### 💡 전체 요약 정리
- **3~4주차**: 데이터 시각화 및 웹 UI 구축 역량 강화 (**Streamlit**)
- **5주차**: 특정 업무(감성분석, 번역 등)를 수행하는 소형 AI 활용 (**Hugging Face**)
- **6주차**: 대화하고 분석하는 범용 거대 AI의 로컬 환경 구축 (**Ollama**)


# 📊 데이터 분석 및 AI 프로젝트 실전 심화 요약

## 1. 데이터 표시 및 인터랙티브 위젯 (2교시)
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | 정적 데이터를 동적 서비스로 전환하는 인터랙티브 요소의 완벽한 제어 |
| **데이터 핸들링** | - `st.dataframe`: 데이터 검색, 정렬 및 `on_select`를 통한 행 선택 이벤트 처리<br>- `st.metric`: 전일 대비 증감(delta) 등 핵심 지표 시각화<br>- `st.table`: 수정이 필요 없는 정적 데이터의 깔끔한 표시 |
| **인터랙티브 제어** | - **사용자 입력**: 슬라이더, 셀렉트박스 등으로 실시간 데이터 필터링 구현<br>- **콜백(Callbacks)**: 위젯 값이 변할 때 스크립트 재실행 전 특정 함수(`on_change`) 선행 실행 |
| **고급 보안/연결** | - `st.connection`: DB 연결 관리 최적화<br>- `secrets.toml`: API 키, DB 비밀번호 등 민감 정보를 안전하게 관리하는 법 |



## 2. 프롬프트 엔지니어링 & 텍스트 분석 (2교시)
| 구분 | 상세 내용 |
| :--- | :--- |
| **학습 목표** | LLM의 답변 품질을 결정짓는 고도화된 프롬프트 설계 전략 습득 |
| **5대 설계 원칙** | 1. **역할(Persona)**: "당신은 전문 분석가입니다" 등 정체성 부여<br>2. **지시(Instruction)**: 명확하고 구체적인 작업 목표 설정<br>3. **형식(Format)**: JSON, Markdown, 표 등 결과물 형태 지정<br>4. **예시(Few-shot)**: 2~3개의 입출력 예시로 정확도 극대화<br>5. **제약(Constraint)**: 출력 길이, 금지어 등 상세 가이드라인 설정 |
| **분석 고도화** | - **구조화된 출력**: JSON 응답 유도로 프로그램 변수에 즉시 반영<br>- **AI 보안**: 프롬프트 인젝션(지시 무시 공격) 방어를 위한 시스템 가드레일 설계 |
