# 2025년 모바일 프로젝트 강의 요약 (2주차 ~ 5주차)

## 1. 디지털 콘텐츠 기획의 개요 (2주차)

### 1.1. 디지털 콘텐츠의 정의 및 산업 동향
* **디지털 콘텐츠**: 아날로그 형태를 **디지털 부호(0과 1)**로 변환하거나 처음부터 디지털 형태로 제작된 문자, 음성, 이미지, 영상 등의 정보 및 내용물을 의미합니다.
* **디지털 콘텐츠 산업**: 디지털 부호로 인코딩하거나 제작하여 유통 및 소비하는 모든 산업을 포함합니다.
    * **특징**: 전통 미디어가 전자 출판, 디지털 음원 등으로 변모하며 산업의 경계가 모호해지고 있습니다.
    * **확장**: 게임, 모바일, 소셜 미디어 등 **ICT 기술이 융합된 컨버전스 형태**로 영역이 지속적으로 확장되고 있습니다.

### 1.2. 디지털 콘텐츠 산업 분류 (주요 6개 분야)

| 분야 | 주요 세부 분류 (예시) | 핵심 내용 |
| :--- | :--- | :--- |
| **디지털 출판** | e-book, 웹 소설, 실감형 | 전자책, 앱북, 온라인 및 모바일 연재 소설 등. |
| **디지털 만화** | 웹툰 | 온라인 및 모바일로 서비스되는 모든 형태의 만화. |
| **디지털 방송/영화** | 모바일 TV, 실감형 | 네트워크를 통한 방송 서비스 및 실감형 기술이 결합된 영화. |
| **디지털 게임** | 온라인 게임, 모바일 게임, 실감형 게임 | 서버 접속, 모바일 기기, VR/AR 등 가상현실 기술과 결합한 게임. |
| **E-learning** | 교과 교육, 직무 훈련, 실감형 | 온라인/모바일 학습 프로그램 (실감형 기술 적용 포함). |
| **디지털 콘텐츠 솔루션** | 저작물/콘텐츠 보호, CMS, CDN | 콘텐츠 제작/편집 솔루션, 콘텐츠 관리 시스템(CMS), 콘텐츠 전송 네트워크(CDN) 등. |

## 2. 소프트웨어 공학 및 프로젝트 수행의 이해 (3주차)

### 2.1. 소프트웨어 공학의 개념
* **소프트웨어(SW)**: 단순한 **원시코드(source code)**를 넘어, **모든 산출물**(자료구조, 테스트 결과 등)과 **각 단계에서 생산되는 문서, 사용자 매뉴얼**까지 포함하는 포괄적인 개념입니다.
* **소프트웨어 공학**: SW 개발 과정에 **공학적인 원리**를 적용하여 **체계적**으로 접근함으로써, 고품질의 SW를 **효율적으로 개발, 운영, 유지보수**하는 분야입니다.

### 2.2. 소프트웨어 개발 생명주기 (SDLC, Software Development Life Cycle)
프로젝트의 계획 단계부터 유지보수 단계에 이르기까지 일어나는 일련의 과정입니다.

$$\text{계획} \to \text{분석} \to \text{설계} \to \text{구현} \to \text{테스트} \to \text{유지보수}$$

### 2.3. 일정 관리 기법
* **간트 차트 (Gantt Chart)**: **바(Bar) 차트** 형태로 프로젝트의 스케줄링, 예산, 자원 계획을 수립하는 기법. 바의 길이가 소요 시간을 나타냄.
* **퍼트 차트 (PERT Chart)**: 태스크 간의 **의존성 관계**를 표현하는 차트. 원은 태스크, 화살표는 의존성을 나타내며, **임계 경로(Critical Path)**를 식별하여 프로젝트 전체 수행 일정을 관리함.

## 3. 프로젝트 기획 및 분석 (4주차: 요구 분석)

### 3.1. 요구 분석의 개념 및 GAP 분석
* **요구 분석**: 프로젝트의 **목표(To-Be)**와 **현 상황(As-Is)** 간의 차이에서 발생하는 **문제(Gap)**를 발견하고 해결하기 위한 활동입니다.
* **GAP 분석 공식**: $$\text{Gap (문제)} = \text{To-Be (목표)} - \text{As-Is (현재 상황)}$$

### 3.2. 요구사항 확정 절차 (RFP 기반)

| 단계 | 주체 | 활동 내용 |
| :--- | :--- | :--- |
| **1단계** | 클라이언트 (고객사) | **제안요청서 (RFP, Request For Proposal)** 작성 |
| **2단계** | 기획자 (회사) | RFP 분석 및 **요구분석서** 작성 (우선순위 포함) |
| **4단계** | 기획자 | 요구분석서 분석 및 **요구사항정의서** 작성 |
| **6단계** | 기획자 | **요구사항 베이스라인 확정** 및 프로젝트 브리프 작성 |

### 3.3. 사용자 조사 방법
* **에스노그래피 조사**: **사용자 여정 지도(User Journey Map)**를 활용하여 사용자가 제품이나 서비스를 이용하는 과정을 추적하고 시각화함.
* **페르소나 모형 (Persona)**: 제품 개발에 체계적으로 사용할 수 있도록 **실제 사용자 유형을 대표하도록 창조된 가상의 인물**을 의미하며, 사용자의 행동 패턴, 동기, 기대 등을 예측하는 데 사용.

---

## 4. 프로젝트 기획 및 분석 (5주차: 정보 수집 및 동향 파악)

### 4.1. 정보 수집의 원칙
정보 수집 시, **필요성, 수집 범위, 출처의 정확성**을 명확히 설정해야 합니다.
* **신선도**: 최대한 **최신의 정보**를 수집해야 합니다.
* **객관성**: 정보를 왜곡하지 않고, 다양한 견해를 경청하며 객관적인 시각을 유지해야 합니다.

### 4.2. 동향 파악 및 분류
* **동향**: 인간의 삶에 영향을 미치는 공통의 화두를 가지고 트렌드를 예측하는 활동.

| 동향 종류 | 지속 기간 | 특징 및 예시 |
| :--- | :--- | :--- |
| **패드 (Fad)** | **6개월 ~ 1년 (단기간)** | 잠깐 유행하다가 사라지는 동향 (예: 특정 댄스 챌린지) |
| **트렌드 (Trend)** | **약 5년** | 유행이 시작된 후 어느 정도 지속되는 동향 (예: 미니멀 라이프 스타일) |
| **메가 트렌드 (Mega Trend)** | **10년 이상 (장기간)** | 지속적으로 유행하며 사회 전반에 큰 영향을 미치는 동향 (예: 고령화 사회, 모바일 중심의 연결 사회) |

### 4.3. 환경 분석 기법
* **거시 환경 분석**: 기업이 통제할 수 없는 외부 환경(정치, 경제, 사회, 문화, 기술, 자연)을 분석하여 장기적인 전략을 수립하는 기법.
* **STEEP 분석**: 거시 환경 분석의 대표적인 방법으로, 다음 5가지 영역을 분석합니다.
    * **S**ocial (사회)
    * **T**echnology (기술)
    * **E**conomic (경제)
    * **E**nvironmental (환경)
    * **P**olitical (정치)
* **5-Forces 분석**: 산업 내 경쟁 강도를 결정하는 5가지 요인(잠재적 진입자 위협, 대체재 위협, 구매자/공급자 교섭력, 기존 경쟁자 간 경쟁)을 분석하여 산업의 매력도를 파악합니다.

### 4.4. 정보 구조 및 레이블링 (서비스 설계 기초)
* **정보 구조**: 정보 제공 방식에 따라 **선형, 계층형, 네트워크 구조** 등으로 나뉨.
* **레이블링**: 내비게이션에 이름을 부여하는 작업으로, **메뉴명, 버튼명, 섹션 타이틀**이 됨.
    * **원칙**: **보편성**, **직관성**, **일관성**, **상호배타성**을 지켜야 함.

# 📝 프로젝트 구체화 (7주차 강의 요약)

프로젝트 아이디어를 실제 화면으로 구체화하는 과정은 **와이어프레임**, **스토리보드**, **프로토타이핑**, **사용성 테스트**의 단계를 거칩니다. 이 과정은 기획자, 디자이너, 개발자 간의 원활한 소통을 돕고, 최종 결과물의 완성도를 높이는 핵심적인 역할을 합니다.

---

## 1. 와이어프레임 (Wireframe)

와이어프레임은 화면의 뼈대를 잡는 초기 설계도입니다.

* [cite_start]**개념**: 화면의 레이아웃, 내비게이션, 인터페이스 요소(버튼, 입력창 등)의 배치를 시각적으로 단순하게 표현한 설계도입니다[cite: 5100, 5101, 5102].
* **주요 목적**:
    * [cite_start]**구조 설계**: 화면의 구성요소 위치를 미리 정의합니다[cite: 5106].
    * [cite_start]**커뮤니케이션**: 기획자, 디자이너, 개발자 간의 공통된 이해를 돕는 도구로 활용됩니다[cite: 5107].
    * [cite_start]**사용자 흐름 검토**: 사용자의 서비스 내 이동 동선을 점검합니다[cite: 5108].
    * [cite_start]**설계 효율화**: 본격적인 디자인 및 개발에 들어가기 전에 구조를 확정하여 수정 비용을 절감합니다[cite: 5109].
* [cite_start]**제작 도구**: 간단하게 **손으로 그리거나**, **Figma, Adobe XD, Balsamiq** 같은 디지털 툴을 사용하여 제작할 수 있습니다[cite: 5103].

## 2. 화면 레이아웃 및 인터페이스 설계

와이어프레임을 바탕으로 화면의 구체적인 구조와 사용자와의 상호작용 방식을 설계합니다.

### 가. 화면 레이아웃 구성

[cite_start]웹사이트나 앱의 화면은 일반적으로 다음과 같은 공통된 요소로 구성됩니다[cite: 5167].

* [cite_start]**헤더 (Header)**: 페이지 상단에 위치하며 로고, 로그인/회원가입 메뉴 등을 포함합니다[cite: 5183, 5186].
* [cite_start]**GNB (Global Navigation Bar)**: 모든 페이지에 일관되게 제공되는 핵심 메뉴 영역입니다[cite: 5211].
* [cite_start]**콘텐츠 영역 (Contents Area)**: 각 페이지의 주요 내용이 표시되는 본문 영역입니다[cite: 5281].
* [cite_start]**푸터 (Footer)**: 페이지 최하단에 위치하며 회사 정보, 이용 약관, 관련 사이트 링크 등을 제공합니다[cite: 5283].

### 나. 사용자 인터페이스 (UI) 설계

사용자가 시스템을 쉽고 편리하게 조작할 수 있도록 돕는 상호작용 시스템을 설계합니다.

* **인터페이스 종류**:
    * [cite_start]**GUI (Graphical User Interface)**: 아이콘, 버튼 등 그래픽 위주의 인터페이스입니다[cite: 5289].
    * [cite_start]**터치/제스처 인터페이스**: 스마트 기기에서 직접 손으로 터치하거나 특정 동작으로 조작하는 방식입니다[cite: 5286].
    * [cite_start]**음성 인터페이스 (VUI)**: 음성 인식을 기반으로 한 상호작용 방식입니다[cite: 5287].
* **UI 설계 원칙**:
    * [cite_start]**메타포**: 현실 세계의 개념을 비유적으로 적용하여 사용자의 이해를 돕습니다 (예: 휴지통 아이콘)[cite: 5288].
    * [cite_start]**직접 조작**: 사용자가 화면의 객체를 직접 제어하고 있다는 느낌을 제공합니다 (예: 드래그 앤 드롭)[cite: 5288].
    * [cite_start]**일관성**: 시스템 전체에 걸쳐 디자인과 조작 방식의 통일성을 유지합니다[cite: 5289].
    * [cite_start]**피드백**: 사용자의 행동에 대한 시스템의 반응을 명확하게 알려줍니다[cite: 5290].

## 3. 스토리보드 (Storyboard) 작성

스토리보드는 각 화면의 상세 내용을 정의하는 구체적인 설계 문서입니다.

* [cite_start]**개념**: 디지털 콘텐츠 기획에서 '화면 설계서'를 의미하며, 각 화면의 구성, 콘텐츠, 기능, 페이지 이동, 동작 방식 등을 상세하게 정의한 문서입니다[cite: 5292].
* [cite_start]**목적**: 디자이너와 개발자를 위한 **작업 지침서** 역할을 하며, 프로젝트 참여자 간의 **효과적인 커뮤니케이션**을 돕습니다[cite: 5293].
* **주요 구성 요소**:
    * [cite_start]**표지 및 개정 이력**: 문서의 기본 정보와 수정 내역을 기록합니다[cite: 5295, 5296].
    * [cite_start]**서비스 흐름도**: 회원가입, 결제 등 주요 기능의 프로세스를 도식화하여 보여줍니다[cite: 5298].
    * [cite_start]**화면 설계**: 각 화면에 대한 고유 ID, 이름, 경로, 와이어프레임, 그리고 각 요소에 대한 상세한 기능 설명(Description)을 포함합니다[cite: 5299].

## 4. 프로토타이핑과 사용성 테스트

설계된 내용을 바탕으로 실제와 유사한 모델을 만들어 테스트하며 문제점을 개선합니다.

### 가. 프로토타이핑 (Prototyping)

* [cite_start]**개념**: 제품 출시 전, 아이디어를 검증하고 사용자 피드백을 받기 위해 만드는 시뮬레이션 모델입니다[cite: 5325].
* [cite_start]**필요성**: 개발 초기에 오류를 발견하여 **시간과 비용을 절약**하고, 이해관계자들과의 **신속한 의사결정**을 돕습니다[cite: 5326].
* **종류**:
    * [cite_start]**저 충실도 프로토타이핑 (Low-Fidelity)**: 손으로 그린 스케치나 **페이퍼 프로토타입**처럼 간단하게 만들어 전체적인 흐름과 컨셉을 검증합니다[cite: 5328, 5329].
    * [cite_start]**고 충실도 프로토타이핑 (High-Fidelity)**: 최종 결과물과 매우 유사한 디자인과 인터랙션을 구현하여 상세한 사용성을 테스트합니다[cite: 5330].

### 나. 사용성 테스트 (Usability Test)

* [cite_start]**개념**: 실제 사용자가 제품이나 서비스를 사용하면서 겪는 경험을 관찰하고 평가하여 문제점을 찾아내는 과정입니다[cite: 5336].
* [cite_start]**목적**: 사용자의 숨겨진 니즈와 서비스의 장애 요인을 파악하여 사용성을 개선하고 만족도를 높이는 것입니다[cite: 5337].
* **주요 테스트 방법**:
    * [cite_start]**씽크 어라운드 (Think Aloud)**: 사용자가 과업을 수행하며 자신의 생각과 느낌을 소리 내어 말하게 하는 방법입니다[cite: 5342].
    * [cite_start]**아이트래킹 (Eye-Tracking)**: 특수 장비로 사용자의 시선 움직임을 추적하여 어느 부분에 집중하는지 분석합니다[cite: 5343].
    * [cite_start]**휴리스틱 평가 (Heuristic Evaluation)**: 전문가가 정해진 사용성 원칙(Heuristics)에 따라 시스템을 평가하는 방법입니다[cite: 5345].
